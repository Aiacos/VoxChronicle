## Sfide

1. **Error isolation ANCORA ricorrente** — Nonostante l'impegno dall'Epic 1 retro, 3/4 storie hanno avuto issue di error handling in code review:
   - Story 2.3: `_processCategory` fire-and-forget senza `.catch()` (H2), EventBus.emit() senza try-catch (non nel report ma pattern simile)
   - Story 2.4: Callback exceptions corrompono stato stream (H4), EventBus.emit() uccide flush loop (M1)
   - Pattern: il dev implementa il happy path ma dimentica di isolare le eccezioni delle dipendenze esterne

2. **Code review trova sempre issue significative** — Tutte le storie reviewate hanno avuto finding HIGH+:
   - Story 2.1: 2 HIGH (test fragili, validazione input mancante)
   - Story 2.3: 4 CRITICAL + 8 HIGH (il piu' problematico dell'epic)
   - Story 2.4: 5 HIGH + 6 MEDIUM
   - Pattern: self-review pre-commit insufficiente

3. **Bug architetturali sottili** scoperti solo in review:
   - `scene:changed` mai emesso da nessun componente (C1 in 2.3) — l'invalidazione cache L1 e' decorativa
   - `capabilities` hard-coded nei decorators invece di delegare al provider interno (C4 in 2.3)
   - Map mutation during iteration in `invalidatePrefix` (C3 in 2.3)
   - `textContent +=` O(n^2) DOM performance in StreamController (H1 in 2.4)

4. **Story 2.2 senza code review formale** — L'unica story senza review strutturata. Le issue sono state trovate retroattivamente nella review di Story 2.3

## Key Insights

1. **Il self-review pre-commit e' il gap principale** — Le code review trovano issue che il dev avrebbe potuto trovare con una checklist strutturata. Non e' mancanza di competenza, ma mancanza di processo
2. **Error isolation richiede enforcement automatico** — Due epic di impegni verbali non sono bastati. Serve una checklist operativa OBBLIGATORIA
3. **TDD riduce drasticamente i finding in review** — Story 2.4 (con TDD) ha avuto 0 CRITICAL vs Story 2.3 (senza TDD) che ha avuto 4 CRITICAL
4. **Il pattern decorator funziona ma richiede attenzione** — La delegazione di capabilities, il non-leak di opzioni interne (skipCache), e il passthrough corretto di stream richiedono review specifica
