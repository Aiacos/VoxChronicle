  <action>
    Write tests first in tests/narrator/RulesLookupService.test.js, then implement the service.

    The service class:
    - Constructor: `(rulesReference, openaiClient, options = {})` — options: `{ cooldownMs = 300000, logger }`
    - `async lookup(question, { skipCooldown = false, signal } = {})` — main entry point
      1. Normalize topic via _normalizeTopic
      2. Check cooldown (skip if skipCooldown=true)
      3. Phase 1: `await this._rulesReference.searchRules(topic, { limit: 3 })`
      4. If no results, fallback: `await this._rulesReference.searchCompendiums(topic, { limit: 3 })`
      5. Phase 2: Create synthesis promise (NOT awaited) via `_synthesize(question, compendiumResults, { signal })`
      6. Set cooldown (unless skipCooldown)
      7. Return `{ compendiumResults, synthesisPromise, topic }`
    - `async _synthesize(question, compendiumResults, { signal })` — calls OpenAIClient.post
      - Endpoint: '/chat/completions'
      - Model: 'gpt-4o' (per user decision — NOT gpt-4o-mini)
      - System prompt: D&D 5e rules expert, cite sources in brackets, 2-3 sentences max, honest about insufficient sources
      - Build excerpts from compendiumResults[].rule.content (capped at 1500 chars each) with citation headers
      - temperature: 0.2, max_tokens: 300
      - Extract answer from response.choices[0].message.content
      - Extract citations from compendiumResults (use .rule.citation?.formatted or build from .rule.source)
      - Return { answer, citations: string[], usage: response.usage }
    - `_normalizeTopic(text)` — lowercase, split on whitespace, filter stop words (how, does, do, what, is, the, rule, rules, for, a, an, can, i, you, work, works, when, if), filter words < 2 chars, sort, join with space
    - `_isOnCooldown(normalizedTopic)` — check Map, return true if within cooldownMs
    - `_setCooldown(normalizedTopic)` — set Map entry to Date.now()
    - `destroy()` — clear cooldown map

    Mock RulesReference and OpenAIClient in tests. Verify the two-phase hybrid pattern works correctly.
