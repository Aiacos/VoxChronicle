    - The onRulesCard callback receives { topic, compendiumResults, synthesisPromise, source: 'auto'|'manual' }
    - Multiple rules questions in one cycle each fire their own independent lookup
  </behavior>
  <action>
    **Tests first** in tests/orchestration/SessionOrchestrator.test.js — add a new describe block for rules lookup integration:
    - Test: rules question in transcript triggers lookup (mock RulesReference.detectRulesQuestion, mock RulesLookupService.lookup)
    - Test: rules lookup runs in parallel with suggestion streaming (verify both called, suggestion not delayed)
    - Test: rules lookup failure does not throw or affect suggestions
    - Test: onRulesCard callback receives correct data structure
    - Test: rules lookup does NOT fire when no rules question detected

    **Implementation** in SessionOrchestrator.mjs:
    1. Add `_rulesLookupService = null;` and `_rulesReference = null;` to instance fields
    2. Add `onRulesCard: null` to `_callbacks` object
    3. In `startLiveMode()`, accept `rulesReference` and `rulesLookupService` from options, store them
    4. In `_runAIAnalysis()`, BEFORE the streaming/analyzeContext block:
       ```javascript
       // Fire-and-forget rules lookup (independent of suggestion cycle)
       if (this._rulesReference && this._rulesLookupService) {
         const detection = this._rulesReference.detectRulesQuestion(contextText);
         if (detection?.isRulesQuestion && detection.extractedTopic) {
           const lookupPromise = this._rulesLookupService.lookup(detection.extractedTopic, {
             signal: this._shutdownController?.signal
           });
           lookupPromise.then(result => {
             if (result && this._liveMode && this._callbacks.onRulesCard) {
               this._callbacks.onRulesCard({
                 topic: result.topic,
                 compendiumResults: result.compendiumResults,
                 synthesisPromise: result.synthesisPromise,
                 source: 'auto'
               });
             }
           }).catch(err => {
             this._logger.warn('Rules lookup failed:', err.message);
             if (this._liveMode && this._callbacks.onRulesCard) {
               this._callbacks.onRulesCard({
                 topic: detection.extractedTopic,
                 compendiumResults: [],
                 synthesisPromise: null,
                 source: 'auto',
                 unavailable: true
               });
             }
           });
         }
       }
       ```
    5. In `stopLiveMode()` / cleanup, set `_rulesLookupService = null` and `_rulesReference = null`

    6. Also add a public method for on-demand queries from MainPanel:
       ```javascript
       async handleManualRulesQuery(question) {
         if (!this._rulesLookupService) return;
         try {
           const result = await this._rulesLookupService.lookup(question, {
             skipCooldown: true,
             signal: this._shutdownController?.signal
           });
           if (result && this._callbacks.onRulesCard) {
             this._callbacks.onRulesCard({
               topic: result.topic,
               compendiumResults: result.compendiumResults,
               synthesisPromise: result.synthesisPromise,
               source: 'manual'
             });
           }
         } catch (err) {
           this._logger.warn('Manual rules query failed:', err.message);
           if (this._callbacks.onRulesCard) {
             this._callbacks.onRulesCard({
               topic: question,
               compendiumResults: [],
               synthesisPromise: null,
               source: 'manual',
               unavailable: true
             });
           }
         }
       }
       ```
