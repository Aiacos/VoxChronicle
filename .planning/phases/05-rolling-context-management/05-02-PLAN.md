    - Test: _estimateTokens returns 0 for null/undefined/empty string
    - Test: setRollingSummary stores summary text accessible during message building
    - Test: buildAnalysisMessages includes rolling summary as system message between verbatim turns and NPC profiles
    - Test: when total tokens exceed budget, lowest priority components are dropped first (next chapter > NPC profiles > rolling summary > verbatim turns > adventure context)
    - Test: system prompt + user request are ALWAYS included regardless of budget
    - Test: with 12K budget and realistic component sizes, all components fit
    - Test: with tight budget (4K), only system prompt + user request + adventure context remain
    - Test: budget applies 10% safety margin (targets 10,800 when budget is 12,000)
    - Test: simulated 180-cycle session with growing summary never exceeds budget
  </behavior>
  <action>
    Modify `scripts/narrator/PromptBuilder.mjs`:

    1. Add new field in constructor: `this._rollingSummary = '';` and `this._tokenBudget = 12000;`

    2. Add `setRollingSummary(summary)` method: `this._rollingSummary = summary || '';`

    3. Add `setTokenBudget(budget)` method: `this._tokenBudget = budget || 12000;`

    4. Add `_estimateTokens(text)` method: `return text ? Math.ceil(text.length / 4) : 0;`

    5. Modify `buildAnalysisMessages()` to enforce token budget:
       a. Build all components as before but track them as named components
       b. Calculate fixed overhead: system prompt + user request (always included)
       c. Apply 10% safety margin: `effectiveBudget = Math.floor(this._tokenBudget * 0.9)`
       d. Variable components in priority order (highest first):
          - Adventure context (RAG or truncated _adventureContext)
          - Verbatim turns (last 5 from _conversationHistory)
          - Rolling summary (this._rollingSummary as system message)
          - NPC profiles (formatted NPC data)
          - Next chapter lookahead
       e. Include each variable component only if adding it stays within effectiveBudget
       f. Log dropped components at debug level: `this._logger.debug('Token budget: dropping ${key}')`

    6. Insert rolling summary as a system message. Position: after adventure context, before verbatim turns (or after verbatim turns, before NPC profiles — use judgment for best prompt flow). Content: `"SESSION HISTORY (summarized):\n${this._rollingSummary}"`

    7. Keep `slice(-5)` for conversation history entries but make it budget-aware — if verbatim turns are dropped, include fewer or none.

    8. Remove or keep `MAX_CONTEXT_TOKENS` for backward compat but the budget system supersedes it.

    Extend `tests/narrator/PromptBuilder.test.js` with `describe('token budget', ...)` block.
  </action>
