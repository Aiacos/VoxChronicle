  <files>scripts/ai/OpenAIClient.mjs, scripts/narrator/SilenceMonitor.mjs, tests/ai/OpenAIClient.test.js, tests/narrator/SilenceMonitor.test.js</files>
  <behavior>
    - postStream yields content tokens from SSE chunks: mock ReadableStream returning `data: {"choices":[{"delta":{"content":"Hello"}}]}\n\n`, expect yield `{ content: "Hello", usage: null }`
    - postStream handles split chunks across reads: buffer incomplete lines, yield only complete `data:` lines
    - postStream stops on `data: [DONE]` sentinel
    - postStream captures usage from final chunk when `stream_options.include_usage` is set
    - postStream throws OpenAIError on non-ok response (e.g., 401)
    - postStream respects AbortSignal (aborted signal throws)
    - postStream bypasses queue and retry (direct fetch, no _enqueueRequest)
    - SilenceMonitor._handleSilenceEvent returns immediately without calling _generateSuggestionFn when _isCycleInFlightFn returns true
    - SilenceMonitor._handleSilenceEvent proceeds normally when _isCycleInFlightFn returns false or is not set
    - SilenceMonitor logs debug message when dropping silence event
  </behavior>
  <action>
    **OpenAIClient.mjs — Add `async *postStream(endpoint, data, options = {})` method:**
    - Does NOT use `_enqueueRequest` or `_retryWithBackoff` — streaming is long-lived and has its own implicit retry (next cycle). Per research anti-pattern guidance.
    - Calls `fetch()` directly with `this._buildUrl(endpoint)`, `this._buildJsonHeaders()`, `method: 'POST'`, `body: JSON.stringify({ ...data, stream: true, stream_options: { include_usage: true } })`.
    - Pass `options.signal` to fetch for AbortController support. Combine with timeout using same AbortSignal.any pattern from `_makeRequest` (lines 554-573).
    - If `!response.ok`, throw `new OpenAIError(...)` with status.
    - Read body with `response.body.getReader()` + `TextDecoder({ stream: true })`.
    - Buffer partial lines. Split on `\n`. For each complete line starting with `data: `: if `data: [DONE]` break, else parse JSON and yield `{ content: json.choices?.[0]?.delta?.content, usage: json.usage || null }` when content is truthy.
    - Wrap in try/finally with `reader.releaseLock()`.

    **SilenceMonitor.mjs — Add cycle-in-flight guard:**
    - Add `_isCycleInFlightFn` property (default null) and `setIsCycleInFlightFn(fn)` setter method.
    - At the TOP of `_handleSilenceEvent()` (before the existing `_generateSuggestionFn` check), add: `if (this._isCycleInFlightFn?.()) { this._logger.debug('Silence event dropped: live cycle in flight'); return; }`
    - This is a synchronous check — no async, no queuing, per user decision "dropped entirely".
  </action>
